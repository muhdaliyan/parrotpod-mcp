## SECTION: ABOUT PARROTPOD
ParrotPod is an advanced no-code AI-powered orchestration platform designed for building, deploying, and managing voice-first AI agents at scale.

Vision:
Built for everyone—from no-code creators to developers. It bridges the gap between complex machine learning models and real-world voice and messaging applications.

Core Capabilities:
- Voice-First Agents: Conversational agents via LiveKit and high-performance LLMs.
- Dynamic Workflows: Real-time automation including WhatsApp and Telegram.
- Telephony-Ready: Assign phone numbers to agents and route calls automatically.
- Developer Focused: Extensible, open-source architecture built for scalability.

Tech Stack:
React (Frontend), FastAPI (Backend), LiveKit (Voice/RTC), Deepgram (STT), OpenAI & Google Gemini (LLM), SQLite3 (DB).

## SECTION: LOCAL SETUP GUIDE
Prerequisites:
- Node.js v18+
- Python v3.10+
- SQLite3

Installation:
1. Clone the repository:
   ```bash
   git clone https://github.com/muhdaliyan/ParrotPod
   cd ParrotPod
   ```

2. Run 'npm run setup:all' from the project root. This installs frontend dependencies, creates a Python virtual environment, and installs backend requirements.

Environment Setup:
Create a .env file in /backend with your keys (LIVEKIT_URL, DEEPGRAM_API_KEY, etc.). Use 'get_env_reference' for a full list.

Launching Servers:
Run 'npm run dev' from the project root.
- Backend: http://localhost:8000
- Frontend: http://localhost:3000
- Voice Agent: Background worker

## SECTION: GET YOUR API KEYS
LiveKit Cloud:
Sign up at cloud.livekit.io. Go to Settings > Keys. Includes one free phone number.

Deepgram:
Sign up at console.deepgram.com. Create an API Key with 'Administrator' role for STT.

OpenAI & Gemini:
- OpenAI: platform.openai.com. Recommended: gpt-4o-mini.
- Gemini: aistudio.google.com. Recommended: gemini-1.5-flash for lowest latency.

## SECTION: ENVIRONMENT REFERENCE
- LIVEKIT_URL (wss://...): LiveKit WebSocket URL.
- LIVEKIT_API_KEY/SECRET: Auth credentials.
- DEEPGRAM_API_KEY: STT access.
- OPENAI_API_KEY/GEMINI_API_KEY: LLM access.
- DATABASE_PATH: Default is ./parrotpod.db.
- FRONTEND_URL: Used for CORS (default: http://localhost:3000).

## SECTION: AGENT MANAGEMENT
Creating Agents:
Use the dashboard to create an agent. Define its name, persona, and role.

Configuration:
- LLM Provider: Use Gemini Flash for the best conversational speed.
- System Prompt: Keep it concise. Define personality and fallback behaviors.
- Telephony: Assign a number in the dashboard; ParrotPod creates the Dispatch Rule automatically.

Workflows:
Enable WhatsApp or Telegram in the agent's Workflows tab to receive instant notifications for captured leads or call completions.

## SECTION: DEPLOYMENT GUIDE (RENDER)
1. Repository: Ensure Dockerfile is at the project root.
2. Service: Create a "Web Service" on Render using the Docker runtime.
3. Env Vars: Add all secrets from your .env to the Render dashboard.
4. SPA Routing (IMPERATIVE): Add a Rewrite Rule (Source: /* | Destination: /index.html | Action: Rewrite) to handle client-side routing.
5. Persistence: Mount a disk at /app/backend/whatsapp_sessions/ to persist linked devices across deployments.

## SECTION: TELEPHONY & VOICE
Numbers:
Purchase numbers via LiveKit Cloud. They sync to your ParrotPod Inventory automatically.

Dispatch Rules:
Tells LiveKit where to route incoming calls. ParrotPod manages these rules as you assign/unassign agents to numbers.

SIP Trunking:
Connect enterprise VoIP systems to ParrotPod via LiveKit's SIP infrastructure for professional telephony.

## SECTION: WHATSAPP & TELEGRAM
WhatsApp:
- Link Device: Scan the QR code in the Integrations panel.
- Persistence: Sessions are saved to backend/whatsapp_sessions/.
- Maintenance: Use 'Reset & Fix' if the QR code fails to load or connection hangs.

Telegram:
- BotFather: Create a bot and get the HTTP Token.
- UserInfoBot: Get your numeric Chat ID.
- Test: Click 'Test' in the Integrations panel to verify your bot can reach you.

## SECTION: TROUBLESHOOTING
- Agent not picking up: Verify worker is active (npm run dev) and number is assigned.
- Latency issues: Switch to Gemini-1.5-flash and Deepgram Nova-2. Shorten the system prompt.
- Gemini Empty Content: Ensure conversation history starts with a 'user' message.
- Render 404 on Refresh: Verify SPA Rewrite rules are active.
- WhatsApp Session Lost: Ensure a persistent disk is mounted on Render.
